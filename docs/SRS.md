# Software Requirements Specification

## Requirement 1
Original: The system should be user-friendly.

Problem:
- Not clear
- Not testable

Improved:
The system shall allow a new user to complete registration within 2 minutes.

---

## Requirement 2
Original: The system shall store student records safely.

Problem:
- Not specific

Improved:
The system shall store student records using encrypted database storage.

---

## Requirement 3
Original: The system should load quickly.

Problem:
- Not measurable

Improved:
The system shall load the dashboard within 3 seconds.

---

## Requirement 4
Original: The system shall allow users to register, login, and manage profile.

Problem:
- Not atomic

Improved:
1. The system shall allow users to register.
2. The system shall allow users to log in.
3. The system shall allow users to update profile.

---

## Requirement 5
Original: The system shall send notifications.

Problem:
- Not specific

Improved:
The system shall send email notifications after successful registration.
